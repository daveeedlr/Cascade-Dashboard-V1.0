Storyboard 06 —
Huffmaster · Project Sword

A different client, a different industry, a completely different set of systems — and the same components you have already seen five times. This board exists to prove one claim: that Cascade Agents is a platform with tenants, not a bespoke build per customer.

Prepared byDavid De los Reyes
ForJohn Cuellar
DateJul 17, 2026
TenantHuffmaster
Product nameSword
ScopePhase 1 POC

Huffmaster runs strike staffing and workforce continuity: jobs of 25 to 50 people, a 600 to 700 person healthcare deal in pricing, surges near 1,000, sometimes with ten days notice. Their systems each work — nothing hands off to the next one without a human retyping, re-notifying, or remembering. Sword is the orchestration layer across that gap. Read this board next to Storyboard 01 and notice how little is new: same shell, same nav, same run list, same approval gate. The connectors changed and the words changed. The product did not get rebuilt.

Frame note

The proposal lists rehire I-9 logic as an open question — "who owns this rule inside Sword?" This frame answers it: the system owns detection, a named human owns the decision.

  • Same approval-gate component as Construgypsum, but the constraint is compliance rather than spend. The gate generalizes.
  • Travel held, not booked — the design directly prevents a known money leak.
  • Override exists, is clearly labelled as accepting risk, and is attributed. Removing the option would just push people off-system.
  • Caught 17 days out instead of at the airport. That's the whole value.

When a system has no API

Essium has no public API, so Sword drives it the way a person does — a scripted browser session behind the same connector interface. The rest of the platform cannot tell the difference: same MCP contract, same scopes, same approval gates. It is slower and more brittle, so it is marked, monitored, and first in line to be replaced if a vendor API appears.

Frame note

Put this beside the Construgypsum connectors screen. Seven completely different systems, pixel-identical panel. That is the platform claim, demonstrated rather than asserted.

  • The RPA fallback is shown honestly — dashed border, marked, with its trade-offs stated.
  • Okta SSO in the top bar is what makes the embedded-surface model realistic on day one.
  • "New system = new connector, nothing rewired" is the answer to the maintenance fear that killed the $500K–$2M point-to-point quotes.

What this board proves

Nothing in Sword is a new product. The nav, the stat cards, the run list, the approval gate, the connector panel and the scope model are the same components serving Construgypsum — a building-materials distributor in Ecuador — and Huffmaster, a US strike-staffing operation. What changed is the connector set and the vocabulary. That is the difference between selling a platform with tenants and quoting a bespoke integration project, and it is precisely the difference that made the $500K–$2M quotes unsellable. A full visual re-skin to Huffmaster's own brand is a token swap on one stylesheet, the same exercise already proven on the Lean Payroll Labs rebuild.

Components reused, unchanged
