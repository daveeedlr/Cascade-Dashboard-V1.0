How to read these

  1. Start with the architecture diagram. It's the system underneath all four boards — the gates every request passes through.
  2. Then pick a user. Each board follows one person end to end, frame by frame, with a note beside every screen explaining the design decision.
  3. Watch the error states. They sit next to the happy path on purpose — a storyboard that only shows the ideal case hides the parts the build has to get right.
  4. Toggle light and dark on any page. The whole set flips on one token layer, which is why re-skinning for another client is a stylesheet swap, not a rebuild.
